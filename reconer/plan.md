## 4. 相机模型与可见面分析

### 4.1 视角几何 — 第三人称镜像

**关键**: 第三人称视角下，面名与选手视角的关系如下：

```
           选手视角                        相机视角 (第三人称)
        +---+                                +---+
        | U |                                | U |
    +---+---+---+                        +---+---+---+
    | L | F | R |      ← 相机在这边 →     | R | B | L |
    +---+---+---+                        +---+---+---+
        | D |                                | D |
        +---+                                +---+
```

所以相机实际看到的是：

| 相机中的位置 | 实际面名 | 可见程度 |
|-------------|---------|----------|
| 正对面 (最大) | **B 面** | ★★★★★ 大面积可见 |
| 画面右侧 | **L 面** | ★★★ 透视缩短，部分可见 |
| 画面左侧 | **R 面** | ★★ 偶尔可见侧边 |
| 顶部边缘 | **U 面** | ★★ 仅边缘 |
| 底部 | **D 面** | ★ 偶尔 |
| 远侧 | **F 面** | ✗ 不可见 |

### 4.2 各动作的可见变化指纹

以 B 面 (主可见面) 和 L 面 (次可见面) 为基准：

| 动作 | B 面变化 | L 面变化 | 可见总变化 | 辨识度 |
|------|---------|---------|-----------|--------|
| **B/B'** | **全 9 格旋转** | 左列 3 格 | 12 格 | ★★★★★ |
| **L/L'** | 右列 3 格 | **全 9 格旋转** | 12 格 | ★★★★★ |
| **U/U'** | 顶行 3 格 | 顶行 3 格 | 6 格 | ★★★★ |
| **D/D'** | 底行 3 格 | 底行 3 格 | 6 格 | ★★★★ |
| **R/R'** | 左列 3 格 | 不变 | 3 格 | ★★☆ |
| **F/F'** | 不变 | 右列 3 格 | 3 格 | ★★☆ |

**关键发现**:
- **B, L** 是最容易识别的 — 整面旋转，变化最显著
- **U, D** 中等 — 两面各一行
- **R, F** 最难 — 仅单面单列，且 R 在 B 面左边缘（常被左手遮挡）

### 4.3 GT 中各动作出现频次

```
F:  3次  (Cross×2, F2L×1)     — 可见变化: L面右列, 辨识度★★☆
L:  9次  (Cross×1, F2L×6, LL×2) — 可见变化: 全面旋转, 辨识度★★★★★ 最好
D:  1次  (Cross)              — 可见变化: 两面底行, 辨识度★★★★ 
U: 14次  (最频繁)              — 可见变化: 两面顶行, 辨识度★★★★
R: 13次  (第二频繁)            — 可见变化: B面左列, 辨识度★★☆ 难点!
f:  2次  (F2L)                — 类似F但多一层, ★★★
```

**风险**: R 出现 13 次但辨识度最低 (★★☆)，是最大挑战。

## 5. 五条技术路线

### 路线 A: 区域颜色直方图差分

**思路**: 不定位 facelet，只看颜色像素面积变化

```
split_N → ROI 内颜色直方图
split_N+1 → ROI 内颜色直方图
差分 → 哪些颜色增减了 → 匹配动作指纹
```

**能力**: 可区分转动面，难区分方向 (U vs U')
**优势**: 最简单，对遮挡容忍度高
**劣势**: R/F 信号弱，方向盲
**降级**: 全局直方图太粗 → 改为分区域 (9 宫格)

### 路线 B: 光流旋转分析

**思路**: 在运动帧中分析像素流动方向

```
运动帧序列 → 稠密光流 → 旋转分量 → 旋转轴+方向+角度
```

**能力**: 直接读取旋转方向 (CW/CCW) 和角度 (90°/180°)
**优势**: 不依赖颜色，可消歧方向
**劣势**: 10 TPS 每动仅 4-10 帧，手指噪声大
**降级**: 稠密光流太噪 → 稀疏光流追踪角点 → 帧差法

### 路线 C: Facelet 精确定位

**思路**: 检测 3×3 网格，逐格读颜色

**能力**: 信息量最大
**优势**: 可精确推理动作+方向
**劣势**: 遮挡时大量格不可读，透视校正难
**降级**: 自动检测失败 → 手动标定第一帧角点 → 光流追踪

### 路线 D: 视频片段分类 (DL)

**思路**: 截取每步动作的 5-15 帧短片，送入分类器

**能力**: 理论上限最高
**优势**: 端到端，不需手工特征
**劣势**: 需要训练数据
**降级**: 真实数据不够 → 3D 渲染合成 → 或 few-shot 迁移

### 路线 E: 魔方约束搜索

**思路**: 利用打乱状态 + 解法结构约束 + 前几步信号，搜索最优序列

**能力**: 可修复个别识别错误
**优势**: 数学最优，天花板最高
**劣势**: 不知打乱状态时受限，计算量大
**降级**: 不知打乱 → 从已识别步骤反推 → 或只在 LL 做模式匹配

## 6. 推荐执行策略: 渐进融合 + 降级决策树

### 6.1 执行优先级

| 阶段 | 路线 | 做什么 | 通过标准 |
|------|------|--------|----------|
| **P1** | A | 颜色差分 → 粗定面 | 能区分 B/L/U/D |
| **P2** | B | 光流方向 → 消歧 | top-1 > 50% |
| **P3** | A+B | 融合 | top-1 > 70% |
| **P4** | E | 约束修复 | 整体 > 85% |
| **P5** | C/D | 兜底长尾 | 按需 |

### 6.2 降级决策树

```
P1 颜色差分
 ├ 有效 → P2 光流方向
 │  ├ 有效 → P3 融合 → P4 约束修复 → 搞定
 │  └ 无效 → 改用帧差法判方向
 │            ├ 有效 → P3 → P4 → 搞定
 │            └ 无效 → 人工标注方向, 训练小模型(路线D)
 └ 无效 → 转 路线 C (facelet)
            ├ 有效 → 直接推理动作 → P4 约束 → 搞定
            └ 无效 → 路线 D (视频分类器)
                      ├ 有效 →搞定
                      └ 无效 → 半自动: 置信度排序 + 人工确认
```

**最终兜底**: 即使全自动失败，有了分段 + 候选动作 + 约束搜索，
用户只需手动确认少数低置信度步骤即可，仍然远优于完全手动复盘。

### 7.2 CFOP 结构约束

```
[Cross]  F L2 D' L' U R' F R2              — 仅 FLDUR 
[F2L-1]  U R' U2' L' U L R' U2' R UD'      — URLUD
[F2L-2]  L F' L' F L U L'                   — LFU
[F2L-3]  U'...f U2' R U R' U2' f'           — UfRU

```

- 全程0次 B 动作 → 候选从 18 降至 15
- Cross 阶段不会有复杂 F2L 手法
- LL 可匹配已知算法库

### 7.3 相邻动作剪枝

- 同面连续 → 合并 (U U → U2)
- 互逆连续 → 消除 (U U' → ∅)
- 对面交换律 → 标准化

## 8. Phase 1 具体实施 (当前)

### 目标

在 split 帧上产出稳定的颜色分布数据。

### 步骤

1. **ROI 标定**: B 面 `(0.30, 0.25, 0.52, 0.70)`, L 面 `(0.52, 0.25, 0.65, 0.60)`

3. **分区统计**: 各面 3×3 子区域 × 6 色像素占比
4. **差分分析**: 相邻帧对差分 → 匹配动作指纹
5. **验证**: top-3 准确率，目标 > 60%

### 10.3 各版本所用特征详解

#### CNN 中间帧 (512d)
- ResNet18 去最后 FC 层, 输入段中间帧的 ROI, 输出 512 维全局特征
- 单独识别率约 40-50%, 是最重要的基础特征

#### HS 颜色直方图差分 (96d) — 当前基线
- 段首帧/末帧的 ROI 分上下两区域, 各计算 HS 联合直方图 (12×4=48 bins)
- 末帧减首帧 = 颜色变化信号, 上下两区域共 96d
- 效果: 对 U 效果好 (顶行颜色变化明显), 对 R/L 效果差

#### 3×3 网格色差 (162d) — 已回滚
- 将 ROI 分 3×3=9 个网格, 每格计算 HS 直方图(6×3=18 bins) 差分
- 比全局直方图多了空间信息, LOO +3.1%, LOVO +4.8%

#### 光流 HOF (72d) — 已回滚
- 段中 2 帧计算光流, 分 3×3 网格各统计 8 方向幅度直方图
- 表达运动方向分布, LOVO +1.4%, 但计算很慢

#### 动态锚点 LR 差分 (2d) — 已回滚
- 用饱和度掩码找"魔方重心 x", 以此分左右区域统计垂直光流幅度
- 理论上 L 转右侧运动大, R 转左侧运动大
- 实际效果微弱 (+0.4%), 因为锚点被绿色外套等污染

### 10.4 关键发现

#### ✅ 已验证有效
1. **CNN + 颜色直方图差分** 是稳固基线, 能达 ~58% LOO
2. **3×3 网格空间划分** 比全局直方图显著提升 (+3-5%), 值得恢复
3. **光流 HOF 有正收益** (+1-2%), 但计算开销大
4. **PCA 降维 + cosine KNN** 效果优于 euclidean

#### ❌ 已验证无效 / 未成功
1. **饱和度掩码定位魔方** — 被绿色外套严重污染 (选手穿绿衣)
2. **帧差热力图定位 B 面** — 手部运动噪声 >> 贴纸信号, 包围盒膨胀到全帧
3. **ROI 三等分 ≈ B 面三列** — 错误假设, B 面只占 ROI 很小一部分
4. **每段帧差质心作为 B 面中心** — 质心被手部运动偏移, 不稳定

#### 🔴 核心瓶颈: L/R 混淆
- L 只有 29 段, R 有 81 段, 3:1 不平衡
- 从相机看: R 转 → B 面**左列**变色, L 转 → B 面**右列**变色
- 但 B 面左/右列在 ROI 中被手部大面积遮挡
- 当前模型把 60-80% 的 L 错判为 R
- **这是准确率突破 65% 的最大障碍**

### 10.7 下一步建议

#### 短期 (提升 L/R 区分)
1. **手动标注 B 面角点** — 在每个视频第 1 帧标 4 个角点, 透视变换得到正面化 B 面, 精确分 3 列
2. **数据增强** — L 只有 29 段, 可尝试左右水平翻转 R 段来增广 L 样本
3. **恢复 3×3 网格色差** — 已验证有效 (+5%), 之前回滚是因为和其他实验混在一起

#### 中期 (突破 70%)
4. **多帧时序特征** — 当前只用首/中/末 3 帧, 丢失了时间信息
5. **分类器升级** — SVM 或轻量 MLP 可能优于 KNN
6. **F 面特征** — F 面完全不可见, 但 F 转会影响 L 面右列和 U/D 面, 需间接推断

#### 长期 (突破 85%)
7. **魔方约束搜索 (路线 E)** — 用置换群验证 + CFOP 结构先验修复低置信度步骤
8. **视频分类器 (路线 D)** — 用 3D-CNN 或 TimeSformer 直接分类短视频片段


## 复盘说明

### 涉及的转动层和转体操作
U,D,L,R,F,r,f,x,y

### 不涉及的转动层和转体操作
B,E,M,S,z

### 注意
本复盘均不存在z转体
x,y转体不计入步数
x转体不会被单独分段, y转体会被单独分段
UD'中间没有空格, 表示这两步是同时开始(或同时完成的), 在分段里它俩同属于一段, 是2个转动
L2'x'中间没有空格, 表示转动和转体是同时开始(或同时完成的), 在分段里它俩同属于一段, 算1个转体

### 在复盘最后加入了可能的额外转体 (不计入分段), 以确保相对于选手最终的朝向是白顶绿前

### 指法记号 (均不算入步数)
↓ = 右手下起手
