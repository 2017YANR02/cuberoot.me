# 人类复盘方法 vs 搜索式复盘

## 一、人类是如何复盘的（未知打乱）

人类复盘的核心策略是 **从后往前逐步倒推**，而不是从前往后逐帧猜测。

### 1.1 锚点：终态已知

无论前面怎么乱，**最后一刻魔方一定是还原的**。这是唯一确定的状态，也是整个推理的起点。

### 1.2 逐步倒推

人类从视频末尾开始，一步一步往前看：

1. **看最后一步**：还原前一帧的魔方长什么样？哪个面刚转过？ → 确定最后一步（例如 `U'`）
2. **在脑中执行逆操作**：已知最后是 Solved，做 `U` 就得到倒数第二步的状态
3. **看倒数第二步**：此时脑中已知的状态和视频画面是否吻合？哪个面变了？ → 确定倒数第二步
4. **重复**：每确定一步，脑中的"虚拟魔方"就更新一步，继续往前推

### 1.3 假设-验证（试错与回溯）

当某一步看不清时：

- **假设**：先猜一个最可能的动作（例如"像是 `R2`"）
- **模拟**：脑中执行逆操作，得到新的预期状态
- **验证**：预期状态和视频画面对不上 → 排除这个假设，换一个（改猜 `R'`）
- **回溯**：如果换了几个还是不对，可能前面某步就猜错了，需要回退更多步

### 1.4 人类用到的线索

| 线索 | 说明 |
|------|------|
| **颜色变化** | 哪些贴纸换了位置 — 最直接的视觉信号 |
| **变化区域** | 只有 B 面右列变了 → 大概率是 L 转 |
| **手部动作** | 右手食指在上面推 → 大概率是 U |
| **CFOP 阶段** | "现在应该在做 F2L" → 缩小候选范围 |

### 1.5 关键总结

> **人类复盘 = 逆序遍历 + 状态模拟 + 假设验证 + 回溯修正**
>
> 人类**不会**试图识别 Triggers（如 Sexy Move），而是**一步一步**地看、猜、验证。
> 人类的"脑内模拟器"是决定性优势——每确定一步，后续的约束就更强。

## 二、搜索式复盘 (Search-Based Reconstruction)

### 2.1 核心思想

将人类的逐步倒推过程形式化为一个 **逆向波束搜索 (Reverse Beam Search)**。

```
人类做的事                    程序做的事
──────────                  ──────────
从还原态开始                  CubeState = Solved
看最后一帧，猜动作              CNN/直方图 → Top-K 概率分布
脑中执行逆操作                 state = move.inverse(state)
和视频对照验证                 计算 Score(预期颜色, 实际像素)
看不对就换一个猜                保留 K 条高分路径 (Beam)
猜错了就回溯                   低分路径被自然淘汰
```

### 2.2 算法流程

```
初始化：state = Solved, t = 最后一段

FOR t = 最后一段 → 第一段：
    1. 获取第 t 段的候选动作 Top-K（来自视觉模型）
    2. 对每条现有路径 × 每个候选动作：
       a. 执行逆操作：state_new = move⁻¹(state)
       b. 计算一致性得分：
          - 视觉得分：state_new 的可见面颜色 vs 视频第 t-1 帧
          - 阶段得分：是否符合 CFOP 结构
       c. 总分 = 路径累积分 + 本步得分
    3. 剪枝：保留总分最高的 Beam_Width 条路径
    
输出：得分最高的路径即为复盘结果
```

### 2.3 为什么逆向搜索优于正向

| | 正向搜索 | 逆向搜索 |
|--|---------|---------|
| **起点** | 未知打乱态 | **已知还原态** ✓ |
| **状态确定性** | 第 1 步就在猜，误差累积 | 第 1 步（末尾）最确定 |
| **可验证性** | 无法验证中间状态 | 每一步都可与视频对照 |
| **CFOP 约束** | 不知道何时进入 LL | LL → F2L → Cross 逐步解锁约束 |

### 2.4 关键模块

| 模块 | 文件 | 职责 |
|------|------|------|
| 魔方状态机 | `cube_state.py` | 维护 54 贴纸，执行转动，查询面色 |
| 视觉模型 | `template_cnn.py` (改造) | 输出概率分布而非硬标签 |
| 一致性评分 | `search_solver.py` | 比较预期颜色 vs 实际像素 |
| 波束搜索 | `search_solver.py` | 管理多条路径，剪枝 |

### 2.5 预期效果

- **单帧分类器 60%** → 搜索后利用上下文和约束，预期 **整体序列准确率 80%+**
- 即使视觉模型将某步的正确答案排在 Top-3 而非 Top-1，搜索仍能通过一致性检查找回
- 核心优势：**错误不再不可逆** — 视觉模型犯的错会在后续步骤的一致性检查中暴露
