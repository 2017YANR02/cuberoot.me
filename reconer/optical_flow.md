# 路线 B: 光流旋转分析 — 详细技术方案

## 1. 为什么选光流

路线 A（帧差颜色分析）失败的根本原因：
**手部运动的帧差信号（30-40% 像素变化）完全淹没了魔方贴纸变化。**

光流的优势：
- **编码运动方向**，而不仅仅是"变了没变"
- 手指平移 vs 贴纸旋转 → 光流方向模式截然不同
- 不依赖颜色，不受皮肤/橙色混淆影响
- 直接区分 CW/CCW（U vs U'），路线 A 做不到

---

## 2. 可用数据

| 参数 | 值 |
|------|------|
| 视频 | 3840×2160, 100fps |
| 每段帧数 | 3-51 帧，中位数 7 帧 |
| 典型动作时长 | 30-100ms (3-10 帧) |
| 暂停段 | 第 27/34 段 (51/49 帧) |
| 最短段 | 3 帧 (第 13/22 段) |

**关键约束**: 3 帧段只有 2 对相邻帧可计算光流，信号极弱。
需要特别关注这些"极短段"的处理策略。

## 3. 每种动作的理论光流特征

### 3.1 坐标系约定

```
相机视角 (第三人称), 图像坐标:
  → x 轴 (向右)
  ↓ y 轴 (向下)

        +---+
        | U |
    +---+---+---+
    | R | B | L |    ← 相机看到的面
    +---+---+---+
        | D |
        +---+
```

### 3.2 光流方向表

| 动作 | 旋转轴 | 可见面变化 | 理论光流方向 | 光流区域 |
|------|--------|-----------|-------------|----------|
| **U** | Y轴(竖直) | B/L 顶行左移 | ← (负x) | 顶部 row0 |
| **U'** | Y轴 | B/L 顶行右移 | → (正x) | 顶部 row0 |
| **D** | Y轴 | B/L 底行右移 | → (正x) | 底部 row2 |
| **D'** | Y轴 | B/L 底行左移 | ← (负x) | 底部 row2 |
| **R** | X轴(水平) | B 左列上移 | ↑ (负y) | 左侧 col0 |
| **R'** | X轴 | B 左列下移 | ↓ (正y) | 左侧 col0 |
| **L** | X轴 | L 面整面旋转 | 旋转 | 右侧 col2 |
| **L'** | X轴 | L 面整面旋转(反) | 旋转(反) | 右侧 col2 |
| **B** | Z轴(前后) | B 面整面旋转 | CW/CCW旋转 | 中央 |
| **B'** | Z轴 | B 面整面旋转(反) | CCW/CW旋转 | 中央 |
| **F** | Z轴 | 不在可见面 | 微弱/无 | L 右缘 |

> [!IMPORTANT]
> U/U' 和 D/D' 的光流方向完全相反 → **可直接区分方向**（路线A做不到的）
>
> R/R' 的光流方向也相反 → **可区分方向**

### 3.3 特征提取思路

将 ROI 分为 3×3 网格，对每个网格计算：
- `flow_x`: 水平分量均值 (正=右, 负=左)
- `flow_y`: 垂直分量均值 (正=下, 负=上)
- `flow_mag`: 光流幅度均值
- `flow_curl`: 旋转分量 (正=CW, 负=CCW)

动作指纹表：

```
U:  row0 有强 flow_x<0, 其他行弱
U': row0 有强 flow_x>0, 其他行弱
D:  row2 有强 flow_x>0, 其他行弱
D': row2 有强 flow_x<0, 其他行弱
R:  col0 有强 flow_y<0, 其他列弱
R': col0 有强 flow_y>0, 其他列弱
B:  中央 有强 curl>0 (CW旋转)
B': 中央 有强 curl<0 (CCW旋转)
L:  col2 有旋转 + col2 flow_mag 大
F:  微弱信号
```

## 4. 算法流程

```
对每个动作段 [split_i, split_i+1]:
  1. 从原视频读取所有帧 frame[split_i] ~ frame[split_i+1]
  2. 裁切 ROI
  3. 对每对相邻帧计算 Farneback 光流
  4. 排除"静止帧" (幅度过小的帧对)
  5. 累积/平均所有帧对的光流
  6. 在 3×3 网格上计算 flow_x/flow_y/curl 特征
  7. 匹配动作指纹表
```

### 4.1 Farneback 光流参数

```python
cv2.calcOpticalFlowFarneback(
    prev_gray, curr_gray,
    flow=None,
    pyr_scale=0.5,    # 金字塔缩放
    levels=3,         # 金字塔层数
    winsize=15,       # 窗口大小 (4K 图可用更大值)
    iterations=3,
    poly_n=5,
    poly_sigma=1.2,
    flags=0
)
```

### 4.2 旋转分量 (curl) 计算

```python
# 光流场 flow = (flow_x, flow_y)
# curl = dflow_y/dx - dflow_x/dy
# 正值 = 顺时针, 负值 = 逆时针
curl = np.gradient(flow_y, axis=1) - np.gradient(flow_x, axis=0)
```

### 4.3 手部运动过滤

思路 1: **不过滤，靠特征区分**
- 手指运动主要是平移（抓握→旋转→放开）
- 魔方面旋转是**局部区域**的**一致方向**运动
- 特征中用"区域一致性"过滤：如果一个网格内方向方差很大→是噪声

思路 2: **ROI 只取魔方中心**
- 缩小 ROI 到只包含魔方贴纸（排除大部分手指）
- 风险：ROI 太小会丢失边缘动作信号（如 U 动作的顶行）

思路 3: **光流幅度阈值**
- 手指大幅运动时幅度很大（>20 px/frame）
- 贴纸旋转幅度适中（5-15 px/frame）
- 取中等幅度区间的流做分析

**推荐**: 先用思路 1（不过滤），观察原始光流特征。不行再加过滤。

## 5. 极短段 (3 帧) 的处理

| 策略 | 说明 |
|------|------|
| **帧间插值** | 不推荐，3帧太少 |
| **累积光流** | 只有 2 对帧，直接叠加 |
| **降低阈值** | 对极短段放宽匹配条件 |
| **上下文推理** | 结合前后段 + CFOP 约束猜测 |

极短段本身就意味着快速转动，光流信号可能反而更强（像素位移更大）。

## 6. 潜在问题

| 问题 | 影响 | 对策 |
|------|------|------|
| 4K 光流计算慢 | 每帧约 1-3 秒 | 先缩小到 960×540 再算 |
| 运动模糊导致光流不准 | 100fps 相对好 | 100fps 已经很高，应该还行 |
| 手指光流干扰 | 遮挡区域方向混乱 | 用方向一致性过滤 |
| 多层同时转 (如 UD') | 两组矛盾信号 | 需要分区域独立分析 |
| F 动作信号极弱 | 可能识别不出 | 依赖约束搜索补全 |

## 7. 实验计划

### 7.1 第一步: 光流可视化

在几个典型动作段上可视化光流：
- 选 U 动作 (第 5 段)、R 动作 (第 6 段)、L 动作 (第 14 段)
- 输出光流方向图 + 幅度热力图
- 判断理论特征是否在实际数据中可观察到

### 7.2 第二步: 特征提取 + 匹配

对所有 45 段计算网格特征，匹配指纹表，报告 top-1 准确率。

### 7.3 通过标准

- 能区分 U/D/R/L/B（不含方向）→ top-1 > 40%（比路线 A 的 11% 好就行）
- 能区分方向（U vs U'）→ 额外加分
- 如果 > 50%，结合路线 E 约束搜索有望达到 80%+
