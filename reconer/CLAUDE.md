# CLAUDE.md

This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## 项目目标

魔方速拧视频自动复盘工具：输入第三人称速拧视频，输出标准 WCA notation 复盘公式（`F L2 D' L' U R' ...`）。准确率目标 ≥80%，允许人工修正剩余部分。设计与实验记录见 `plan.md` / `roadmap.md` / `human.md`（均为中文，是理解本项目的必读背景）。

## 相机几何（最关键的非直觉前提）

第三人称机位与选手视角是**镜像**关系。相机主要看到的是 **B 面**（正对、大面积），不是 F 面：

| 相机中位置 | 实际面名 | 可见度 |
|-----------|---------|--------|
| 正对（最大） | B | ★★★★★ |
| 画面右侧 | L | ★★★ |
| 画面左侧 | R | ★★ |
| 顶/底边缘 | U / D | ★★ / ★ |
| 远侧 | F | 不可见 |

推论：相机的左 = B 面的右列（索引 47,50,53），相机的右 = B 面的左列（索引 45,48,51）。`greedy_reverse.py` 里 B 面 3×3 提取做了这个镜像重排。**L/R 混淆 + 数据不平衡（R:81, L:29）是准确率的头号瓶颈。**

## 三步流水线

```
视频 → [motion_detect] 分段 → [template_cnn] 分类出概率分布 → [greedy_reverse] 逆推 → 复盘公式
       Step 1              Step 2                          Step 3
```

开发时用手工标注的 `*.splits.txt` 替代 Step 1 输出，隔离误差。核心洞察（`human.md`）：**从 Solved 态逆序倒推**，因为末帧状态唯一确定，每一步都可与视频对照验证 —— 逆向搜索优于正向逐帧猜测。

## 数据格式（`videos/`）

- `*.MP4` — 原始 4K/100fps 视频，单个 500–800MB，已 gitignore
- `*.splits.txt` — **Ground Truth**。首行 `Splits:679:686:...`（帧号，N+1 个点对应 N 段）+ 复盘 notation 行（每个 token 是一段的标签）。`template_cnn.py` 和 `greedy_reverse.py` 都靠这个文件对齐
- `*.probs.json` — 每段的 Top-K 面概率分布（Step 2 输出，Step 3 输入）
- `features_cache.npz` — 608d 特征缓存。**改视频/改 ROI 后必须手动删除重建**，否则用的是过期特征

## Notation 约定（见 `plan.md` 末尾「复盘说明」）

- 涉及：U D L R F r f x y；**不涉及 B E M S z**
- CubeState facelet 顺序：U(0-8) R(9-17) F(18-26) D(27-35) L(36-44) B(45-53)；颜色 id：U=0 R=1 F=2 D=3 L=4 B=5
- `x` 转体不单独分段（与前一动作捆绑，如 `L2'x'` 算 1 段）；`y` 单独分段
- `x y` 转体不计入步数
- `UD'`（中间无空格）= 同时开始/完成，属同一段但算 2 个转动
- `↑↓·` 是指法记号，`...` 是卡顿，解析时都要剥离（`splitByFingering`）
- splits.txt 末尾的 `y2` / `z2` 等是用户加的朝向校正，不对应视频段（`greedy_reverse.parseGT` 里作为 `tailRotations` 分离）

## 当前状态与陷阱

### 引擎 (物理语义, 2026-07-06 完成, 已验证)

- **`src/rotation-perms.ts` + `src/anchored-search.ts`**: 三种语义并存, 辨析见 rotation-perms.ts 头注 — ① canonical (cube-state 逐 token apply, 转体=恒等, r≡L); ② conjugated (cube-state 整串 apply); ③ **physical (转体是真实贴纸置换, 中心块动) — 搜索/回放/观测一律用这套**。转体置换经 162 条共轭恒等式对 functions.cpp 面映射表验证。
- **oracle 结论 (5 GT 视频, 仿真观测)**: 物理语义 + 逐段 B 面观测 (噪声 12.5%) → **5/5 锚定, 逐段 99.1%, 4/5 编辑距离 0**(Phase-1 canonical 只有 87.3%): 中途 y'、宽转、U'D、x 捆绑全部精确还原。跑法: `npx tsx scripts/oracle-eval.ts --beam 2048 --mode vis`; 面身份边缘化 + 随机 κ + B/U 漂移版: `--mode visface` (B/U 限面 **5/5 锚定 92.1%**, `--allfaces` 关限面对照 65.9%)
- **敏感性包络 (定提取器 KPI)**: 噪声 0.25 → 5/5 99.1%; 0.35 → 1/5 崩。**整段丢观测是头号杀手**: dropout 0.4 → 0/5 (哪怕格子全对); 每段只见 5/9 格但全覆盖 → 3/5 90%。→ **提取器 KPI: 每边界覆盖率 > 单格准确率 (但准确率也要 ≥75%)**。
- 锚定 ≠ 正确: 短序列存在"另解"碰巧锚定 (如 R F2x' F2x' u'), 靠观测得分区分。y+U/D 同轴组合有等价写法 (R U y' R U ≡ R d R U), 搜索可能输出更紧凑的宽转写法。

### 相机约定 (2026-07-06 金标验证 + 共轭语义修正, 单测锁死)

- **无列镜像**: 相机 3×3 行主序直接对应 B 面 facelet 45..53 同序 (video3 帧 681 逐贴纸金标)。`bface-color.ts` 的列镜像是 Python 旧布局的约定, **勿沿用**。
- **共轭分解 (最关键)**: 相机态 = ρ⁻¹∘R_t∘ρ (GT 系与相机系差常数旋转 ρ)。拆两半:
  - **内侧 (位置重排)**: 被"指派边缘化"吸收 — 合法指派集 = **24 个朝向 ρ 的 B 窗口回拉** `assign[i]=ρ[45+i]` (`FACE_ROT_ASSIGN`)。**勿用 facelet 下标网格 rot90 硬造** — 各面下标布局从外侧看未必行主序, 硬造过一次, 真实数据得分≈随机才查出来 (两种指派集在本状态机布局下恰好数值等价, 但推导必须走窗口回拉)。
  - **外侧 (常数颜色重标 κ, video3 = y2: G↔B/O↔R)**: **不被指派吸收**! 搜索靠 24 朝向初始 beam 吸收 (真路径 S'_t = ω∘R_t, ω=κ), 因此**锚集必须左复合 {ω∘打乱态}** — 右复合 {打乱态∘o} 只在 κ=id 的纯仿真里与之相交, 真实数据下不含真路径 (潜伏 bug 已修, `test/anchored-search.test.ts` κ≠id 用例锁死)。离线指标里 κ 按视频拟合 (24 候选全局最优, `real-eval.ts bestAssign`)。
- **面身份随持握漂移**: 起手慢速期相机看 B 面, 从第 2-3 步起选手前倾, 相机主要看 U 面 (斜视角)。观测模型**面身份边缘化** (`RawFaceObs`/`logRawObs`, 指派集可 `rawFaces` 限面)。oracle `--mode visface` 仿真此设定 (含随机 κ + B/U 漂移)。
- **限面收益 (oracle, 12.5% 噪声, 随机 κ)**: 全 24 指派边缘化 65.9% → B/U 限面 (8 指派) **92.1%, 5/5 锚定** — 边缘化的证据稀释大部分可由持握先验收回。

### 真实提取 (WIP, `src/sticker-blobs.ts` + `scripts/dump-frames.ts` / `real-eval.ts` / `debug-blobs.ts` / `diag-*.ts`)

- 管线: 全程连续 framedump (960×540) → `activityMask` (跨帧稳定性掩码) → 同色连通域色块 → 反光环带重标 (W 块 1.25× 壳层放宽饱和色相投票) → 多色簇 + 尺寸一致性过滤 → RANSAC 仿射基网格 (斜面门槛: |sin|>0.4, det>0.12·medNN²) + **Gauss 基约减** (幺模歧义, 不约减 3×3 窗口会斜) → 顺序多晶格 (≤2 面/帧, 拟合后移除内点再拟合) → 格心采样 → **时间连续性跟踪** (`trackFaceGrid`: 冷检测失败帧用上帧晶格作先验, 平移搜索 + 直接格位采样, 5 连败杀先验 — 逐帧读出率 25%→90%+) → **静止链检测** (跨帧网格一致 ≥2 帧成链, ≤1 格容错 + 空帧桥接 ≤2) → 链共识网格按中心帧归属所在段。
- **搜索观测 = 双端点语义** (`anchored-search.ts rawObservations` 支持每段多网格): 段内任意时刻读数必属段起点态或终点态之一 (拧转前/后; 拧转中非转动层两者皆合), 每网格 max(起点,终点) 段内求和 — 标注滞后 (split=手部动作起点, 拧转滞后 0~8 帧) 的 ±1 归属歧义自动消解; y 段的链归前一非转体段 (y 是整机旋转, 面边缘化天然吸收)。oracle 回归 92.1% 不受影响。
- **观测按静止链而非逐边界强读的原因 (踩过的坑)**: 逐边界就近强读会把**邻段状态**当本段观测 (同一可读帧被归到多个边界), 对搜索是毒药 (比 dropout 更狠); 双端点语义 + 链化从源头消解。
- **现状数字 (2026-07-06 深夜, 诚实)**: 跟踪后逐帧读出率 90%+ (冷 25% + 跟踪 65%), 段有观测率 86.9%; **帧级软覆盖 ±12 达 60%** (信息在流里, 首次踩到包络线); 但**链级准确率只有 ~55% (乱态底噪 48%), SNR 太薄** — 端到端 0/5: 弱观测是负资产 (全喂 8.3% < 门控 len≥4&填≥7 11.4% < 早期少而准 17.5%, 逐段准确率均为未锚定垃圾路径口径)。GOOD 链 (≥85% 匹配) 每视频只有 1-5 条, 无 GT 时挑不出来。**结论: 启发式 CV 已到平台期, 破局点 = 把链级准确率从 55% 提到 75%+ (噪声悬崖), 见 prior_art.md 蓝图**。
- **瓶颈二维分解 (2026-07-06, `real-eval.ts --oraclesingle/--oraclefull/--oracleobs` 网格切, 全 5 视频)**: 用真实覆盖模式 + 可控注入精度的 oracle 观测, 把"覆盖密度"与"逐格精度"两轴解耦 —— ① 真实稀疏覆盖(25/40 边界, 8/40 双端) + **完美精度** → 锚定失败 12.5% (覆盖不足杀); ② 满覆盖(每段单主导面 or 满 B+U) + **完美精度** → **5/5 锚定 90.8%** (引擎本身没问题, 全 5 视频泛化); ③ 满覆盖 + **真实精度 0.62** → **0/5 锚定 10%** (精度在噪声悬崖下方, 全 5 视频泛化); ④ 满覆盖 + 0.62 + **pin(钉死指派)** → 更差 2.5%。**结论: 锚定要覆盖×精度联合达标 (覆盖≈全段 且 逐格≥~75%), 改单轴无用**; 真实管线两轴都欠 (覆盖 25/40 + 精度 0.60), 与 oracle 敏感性包络的噪声悬崖 (0.25→5/5, 0.35→崩) 一致。
- **负结果 (勿重试)**: ① **pin (每路径固定指派隐变量, `rawPinWindows`)** 在噪声下反伤 (0.62 满覆盖 10%→2.5%): 固定隐变量把状态空间 ×16, "错态+碰巧对齐 pin"组合爆炸快于它锐化 SNR, beam 被稀释; drift 变体同理。② **junk 混合鲁棒似然 (`rawJunkProb`, robust logaddexp 地板)** 救不了 60% 精度: 60% 观测不是均匀垃圾 (地板抓不到), 而是系统性偏向似是而非的错误态。③ **perframe 密集逐帧喂搜索 (`--perframe`, visualWeight 0.35)** → 2.5%, 比链式 12.5% 更差: 密集但噪声, 邻帧误分类高度相关不能靠平均消 (印证"弱观测是负资产")。④ **forward 正向搜索 (从打乱锚起, `--forward`)** GT t=0 即死 (未根因, 疑实现或早段证据比设想更弱)。⑤ **每视频静态颜色自标定 (`src/color-calib.ts`, `--calib`/`--calibgt`)** 修不动精度轴: 已知态帧 (打乱/复原/GT 全段) 反标 5682 格样本, 色度盘+亮度特征上逐类对角高斯的样本内天花板仅 65-70%, **LOO 7-NN (无参, 最乐观口径) 也只 69.9-77.6% (合计 73.0%)** — 全在 75% 悬崖下方。混淆是视角/光照条件性的 (R/O/Y 压缩在色相 6/16/32 半度窄带, 反光洗白), 单格颜色特征**本质饱和**, 任何静态分类器/调阈值都无解, 别再试。管线级标定重提取实测还略降 (60.0%→57.8%)。⑥ **软观测 (逐格 6 色概率向量, `RawFaceObs.probs` + `--oraclesoft`)** 同精度下反而大败: 满覆盖 + 真实特征噪声 (GT 样本池抽样) 下 **0/5 锚定 ~5.6%**, 而硬观测均匀噪声 @0.70 有 1/5 锚定 48.9% — 高斯错拟合的"自信押错"格主动喂错误路径, 比硬 miss 的有界惩罚更毒; 回火 τ 数学上 ≡ 全局证据缩放 (逐格常数偏移在假设间相消), 修不了符号错误, 别扫 τ。**推论: 均匀噪声 oracle 低估了难度 — 真实偏向性错误 (R→O 系统性) 下真墙比 75% 更高**。⑦ **每观测光照增益校正 (`scripts/gain-ceiling.ts`, render&compare 内环的真标签最优模拟)** 全 5 视频一致反伤 (LOO k-NN 77.3→74.7 / 73.9→69.2 / 70.3→63.6 / 70.6→64.5 / 70.3→66.0): 混淆**无每帧相干光照成分**, 是每格级的 (贴纸角度镜面反光/运动模糊/局部映照) → **render&compare 的颜色残差通道同样封顶**, 其剩余价值只在几何/时间平滑/亚格结构, 投入前须重估。⑧ **手部关键点第二证据通道 (`hands/`, MediaPipe HandLandmarker, 2026-07-09)** 在本素材上无段级信号: 检测率 91-95% 双手全程在框 (提取无忧), 但 6-way face 分类 LOVO 只 ~48-53% (probs 旧链 57.6%, 墙 75%); **机理: 9 TPS 下双手永远都在动** — r_energy/主动手全类平坦 (峰速手 93% 恒同一只), "哪只手拧的"这个人类直觉信号在段窗粒度不存在; 镜像增广 (L↔R 对称)、world-landmark 3D Kabsch 旋转轴、魔方格坐标投影各 +≤1pt; **视频内 k-fold ≈ LOVO (50±2%) → 是特征不携带信号, 不是跨视频漂移**; 与 probs 融合反降 (w=0.7 56.8% < 纯 probs 58.1%) — 错误相关, 叠不出增量。手部 LSTM 别指望: felikemath 82% 是 3 类慢速场景, 与本素材 6 类 9TPS 链式指法不可比。⑨ **晶格带光流/位移三连败 (`hands/flow_features.py` 三版)**: Farneback 稠密光流死于拧转帧大位移+运动模糊 (能量渗邻带) 与快转晶格跟踪滑脱 (可视化实锤: 格点悬在手上); 相邻帧对条带块匹配死于**灰度 1 格周期孔径** (炫彩贴纸灰度近同亮, 条带只剩黑缝, NCC 在 0/±1/±2 格同优); 静止对静止 RGB 条带死于**面外流入** (B 面上 U 转不是"内容沿条带滑 1 格" — 流入贴纸来自侧面, 仿射晶格延长线出面即折到别的 3D 平面) + 条带一半是手指 + U 面斜条透视过冲采到背景。带级指纹直检 (|d| 冠军带 per face) 每版都散, 分类器 LOVO 全程 47-53% 平台。⑩ **贴纸外观守恒动作打分 v2 (`scripts/conserve-eval.ts`, 2026-07-10)**: 静止链共识 + 色度-only + B/U 双视图 + 全局指派拟合 + 平移边缘化全套做完, 真实 face top1 仅 26.7% (覆盖 56%, 门槛 probs 57.6%)。**打分机器无罪** (神谕阶梯: 全覆盖合成观测 97.6%/100%, 真实覆盖+GT 内容 75-77%), 三个提取层根因 (同态自检实锤, 详见脚本头注): (a) **晶格重获取平移漂移** — 同边界跨链对齐 p50 -30, 加 ±1 格平移搜索 → -10, 而链内 split-half 自距 p50 -1~-5 (特征本身稳): 重获取的 3×3 窗常错位 1 格读到邻位贴纸;
