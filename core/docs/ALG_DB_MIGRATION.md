### Schema(从 `shared/src/alg.ts` 读的接口)

```typescript
interface AlgFile {
  scrapedAt: string;     // ISO timestamp
  source: string;        // URL 或本地路径(speedcubedb / docx)
  puzzle: string;
  set: string;
  cases: AlgCase[];
}

interface AlgCase {
  name: string;          // "Aa", "Solved Pair I"
  subgroup: string;      // "Adj Swap", "Solved Pair", "" (无子组)
  setup: string;         // 打乱公式
  standard?: string;     // 标准公式(可选;1lll/zbll 没有)
  sticker: AlgSticker;   // polymorphic(face / f2l / raw),适合 JSONB
  algs: AlgEntry[][];    // 2D: F2L 4 个 orientation × N 备选,其它 1 × N
  oriNames?: string[];   // F2L 才有(4 个 orientation 名)
  trainerKey?: string;   // ZBLS 才有(链向 trainer 模块)
}

interface AlgEntry {
  alg: string;
  altId?: string;        // speedcubedb 上的备选 id
  ytId?: string;         // YouTube finger-tricks tutorial id
}

type AlgSticker =
  | { kind: 'face'; us: string; ub: string; uf: string; ul: string; ur: string }
  | { kind: 'f2l'; fl: string }
  | { kind: 'raw'; tag: string; attrs: Record<string, string> };
```

### Client 唯一入口

**只需要改这 1 个函数**,client 其它代码 0 改动。

## 1. 决策点(待用户确认)

| 项 | 推荐 | 备选 |
|---|---|---|
| **存储** | 整个 JSON `cases` 字段塞 JSONB(每个 set 1 行) | 拆 `alg_sets` + `alg_cases` 两表(每 case 1 行) |
| **GitHub Pages 兼容** | GH Pages 调 `cuberoot.me/api/alg/...`(跨域,CORS 已允许) | 保留 JSON 作 fallback |
| **数据 source of truth** | 迁完后 **JSON 文件删除**,DB 唯一 | 保留 JSON 作冷备(同步麻烦,易漂移) |
| **API 路径** | `GET /api/alg/sets`(列表)+ `GET /api/alg/sets/:puzzle/:set`(单个 set 全部 cases) | RESTful 拆细 |
| **缓存策略** | API 返 `Cache-Control: public, max-age=3600`(1h),浏览器缓存 + ETag 触发更新 | 强 max-age=86400(24h) |
| **admin 编辑入口** | case 卡右上 admin-only 铅笔按钮,弹出"编辑 case"表单(改 caseName / setup / standard / 加 alg / 改 sticker) | 单独 admin 后台页 |

### 推荐理由(每行展开)

1. **整 JSONB vs 拆两表**:推荐**拆**(`alg_sets` + `alg_cases` 两表)。优点:
   - admin 改单个 case 只 UPDATE 一行,不用 read-modify-write 整个 set 的 JSONB
   - 加 community submissions / case 关联性更自然
   - 行数 < 10k(所有 set cases 加起来),性能没顾虑
   - 保留 case 顺序通过 `position` 列

2. **GH Pages 调 cuberoot.me API**:CORS 在 server `index.ts` 已经允许了 `https://ruiminyan.github.io`,跨域可以工作。GH Pages 是镜像,主流量在 cuberoot.me,这样最简单。

3. **JSON 删除**:留着只会"两份都改"出错;迁完一刀切。如果以后想 GH Pages 离线工作,可单独写一个 build-time 脚本从 PG dump JSON 进 SPA bundle —— 但本期不做。

4. **API 路径**:GET 整个 set(跟现有 JSON 行为 1:1)最简单 client 改动 = 0。

5. **缓存**:1h 是 admin 改 case 后用户最多等 1h 才看到。强缓存 24h 更省流量但更新延迟大。1h 平衡。

## 2. PG Schema

```sql
-- alg 公式库 source-of-truth(替代 41 个 JSON 文件)
CREATE TABLE alg_sets (
  puzzle      VARCHAR(20) NOT NULL,
  set_slug    VARCHAR(50) NOT NULL,
  source      TEXT,
  scraped_at  TIMESTAMPTZ,
  updated_at  TIMESTAMPTZ NOT NULL DEFAULT NOW(),
  PRIMARY KEY (puzzle, set_slug)
);

CREATE TABLE alg_cases (
  id          BIGINT GENERATED BY DEFAULT AS IDENTITY PRIMARY KEY,
  puzzle      VARCHAR(20) NOT NULL,
  set_slug    VARCHAR(50) NOT NULL,
  position    INTEGER NOT NULL,        -- 保持 JSON 数组里的原始顺序
  case_name   VARCHAR(128) NOT NULL,
  subgroup    VARCHAR(64) NOT NULL DEFAULT '',
  setup       TEXT NOT NULL DEFAULT '',
  standard    TEXT,                    -- nullable(1lll/zbll 没有)
  sticker     JSONB NOT NULL,          -- polymorphic { kind, ... }
  algs        JSONB NOT NULL,          -- 2D AlgEntry[][]
  ori_names   JSONB,                   -- F2L 才有(string[])
  trainer_key VARCHAR(32),             -- ZBLS 才有
  updated_at  TIMESTAMPTZ NOT NULL DEFAULT NOW(),
  FOREIGN KEY (puzzle, set_slug) REFERENCES alg_sets(puzzle, set_slug) ON DELETE CASCADE
);
CREATE INDEX idx_alg_cases_set ON alg_cases(puzzle, set_slug, position);

-- updated_at 触发器(已有 trg_set_updated_at 函数,本期 schema.pg.sql 已建)
CREATE TRIGGER alg_sets_updated_at BEFORE UPDATE ON alg_sets
  FOR EACH ROW EXECUTE FUNCTION trg_set_updated_at();
CREATE TRIGGER alg_cases_updated_at BEFORE UPDATE ON alg_cases
  FOR EACH ROW EXECUTE FUNCTION trg_set_updated_at();
```

> **关键设计点**: `(puzzle, set_slug, case_name)` **不**加 UNIQUE 约束 —— ZBLS 的 Geng 86 cases 可能跟标准 case 重名,只通过 subgroup 区分(且 subgroup 也可能重)。位置由 `position` 列保证。

## 3. API 设计

### `GET /api/alg/sets`
列出所有可用 sets(给客户端做 catalog 索引,代替 ALG_CATALOG 静态常量?)。本期不动 ALG_CATALOG(它带显示名 + umbrella 标志,值得 stay code-side),只返 (puzzle, set_slug) pairs 用作可达性校验。

返回:
```json
[
  { "puzzle": "3x3", "setSlug": "pll", "scrapedAt": "...", "updatedAt": "..." },
  ...
]
```

### `GET /api/alg/sets/:puzzle/:set` ⭐ 核心
返回**完整 AlgFile** JSON 形态(跟现有 JSON 文件 1:1 同构):
```json
{
  "scrapedAt": "...",
  "source": "...",
  "puzzle": "3x3",
  "set": "pll",
  "cases": [
    { "name": "Aa", "subgroup": "Adj Swap", "setup": "...", "standard": "...", "sticker": {...}, "algs": [[...]] },
    ...
  ]
}
```
header: `Cache-Control: public, max-age=3600`, `ETag: <hash of updatedAt>`

### `PUT /api/alg/sets/:puzzle/:set/cases/:id` (admin only)
改单个 case 的 caseName / subgroup / setup / standard / algs / sticker。body:
```json
{ "caseName": "...", "subgroup": "...", "setup": "...", "standard": "...", "sticker": {...}, "algs": [[...]] }
```

### `POST /api/alg/sets/:puzzle/:set/cases` (admin only)
新增 case(给当前 set 加一行,append 到末尾 position)。

### `DELETE /api/alg/sets/:puzzle/:set/cases/:id` (admin only)
删 case。

## 6. 风险与对策

### R1: 1lll 74k 行 JSON 太大
- **现象**: PG 单表 8MB(1lll ~3915 cases,每 case 几 KB)。`GET /api/alg/sets/3x3/1lll` 返回 ~8MB JSON。Vite 现在按需 lazy-load,如果用户开 `/alg/3x3/1lll` 也会下载完整 JSON,行为没差。
- **对策**: 加 gzip(Hono `gzip()` middleware 或 nginx 反代加 `gzip on`)→ 8MB 压成 ~1MB。

### R2: GH Pages 跨域调 cuberoot.me 卡顿
- **现象**: GH Pages 用户从 ruiminyan.github.io 加载 alg 时,要走 CORS preflight + 跨域请求,首次访问慢。
- **对策**: cors preflight 是浏览器自动,不会卡;CDN 不影响(API 直连 origin)。如果 cuberoot.me 挂,GH Pages 上 alg 模块挂(可接受 —— 那时主站也挂)。

### R3: admin 改 case 后,其它用户 1h 内看到旧版
- **现象**: API 缓存 1h。admin 改完后想立即看到,得手动 hard refresh。
- **对策**: 改 case 后 server 返 `ETag` 变了,浏览器会重新拉。也可以让 client 在 admin 编辑成功后主动 refetch / 强制 cache bust(`?v=<timestamp>`)。

### R4: 历史 ZBLS Geng 86 cases 重名导致 subgroup 切换混乱
- **现象**: ZBLS umbrella 模式按 subgroup 分组渲染,如果同 set 内有重名 case 跨 subgroup,UI 可能困惑。
- **对策**: 这是当前 JSON 已存在的现实,迁 DB 不引入新问题。后续 admin 用编辑 UI 把 Geng 86 cases 整理到正确 subgroup 即可。

### R5: 备份与回滚
- **现象**: import 脚本写错把数据搞坏。
- **对策**: 41 个 JSON 文件是 git repo 里的不可变备份。任何时候可以重新 import。`alg_cases` 表也可以 `pg_dump -t alg_cases` 增量备份。
