# /sim 手部指法规格(FINGERING)

**单一权威来源**:所有「哪只手、哪根手指、以什么方式做哪一步」的规格记在本文档。
改指法 → 先改这里 → 再改 `handsRig.ts` 的 `classifyHandGesture`(映射)与 `applyHand`(编排)→ 同步 `tests/hands_gestures.test.ts`。
穿模禁令 / oracle / 标定方法见 memory `project_sim_hands_rig`(不在本文档重复)。

坐标约定:标准配色正视,F=前(z+)、B=后(z−)、U=上(y+)、D=下(y−)、R=右(x+)、L=左(x−)。
角块记号如 UFR = U/F/R 三面交角。单位 U = 世界单位(SIZE=64 时棱长 192、半宽 96)。

## 1. 握持状态(grip)

| 名称 | GripName | 记号 | 定义 | 右手指位(左手 x 镜像) |
|------|----------|------|------|------------------------|
| 中手 | `home` | `·` | 基准握 | 拇指压 FR 棱 F 面,食指 BUR / 中指 BR / 无名指 BDR 贴 B 面,小指悬空收拢 |
| 上手 | `up` | `↑` | home 绕 x 轴 +90°(= R 提交后的腕姿) | 拇指起手在 U 面,指列落 D 面 |
| 下手 | `down` | `↓` | home 绕 x 轴 −90° | 拇指压 D 面(DR 缘),**食指落 U 面 UFR 区 / 中指 UR 棱 / 无名指 UBR 区**,指腹朝下压 U 面 |

- 记号占一步,只动手不动魔方(`regrip()` 动画 / `simulateGrips` 静态推演)。
- 组合名:**双中手** = 两手都 `home`;**左中右下** = 左 `home` + 右 `down`;其余类推。
- x 轴单层 weld(R/L 族)提交会把旋转烘进该手 grip(手停在转完的位置)——「上手」恰等于 home 被 R(或左手 L')提交后的姿态。

## 2. 记号汇总(解法框)

| 记号 | 含义 | 解析 |
|------|------|------|
| `↑` `↓` `·` | 换握(上手 / 下手 / 回中手) | 独立步,PlayerControls `GRIP_MARKS` |
| `p` 后缀 | **推法**(push):该步用「指尖推层」指法而非默认拨法 | 紧跟在招式 token 后,如 `U'p`;喂 cubing.js 前剥除,对魔方状态零作用 |

**`p` 记号定义(2026-07-08 本文档发明)**:
- 语法:招式 token 尾接小写 `p`,与招式之间无空格(`U'p` ✓;`U' p` ✗)。助记:push / 推 / 平推。
- 当前仅定义 **`U'p`**:右食指推 U'(见 §4.5)。镜像扩展 `Up`(左食指 LUB→RUB)保留但未编排。
- 其它招式带 `p` = 未定义:解析时剥掉后按默认指法播放(不报错),便于向后兼容将来扩展。
- 剥除函数与 `stripGripMarks` 同层处理:invert / mirror / 反推打乱 / recon 手递前都必须剥。

## 3. 默认手指分配总表(`classifyHandGesture`)

层类别:high = 含最外层(R/U/F 族),low = 含 0 层(L/D/B 族),mid = 中层(M/E/S),whole = 整体转。
dir = sign(层角,绕引擎轴 AXIS_VEC);下表以「招式记号」为准。

| 轴 | 招式 | 手势 |
|----|------|------|
| x | R 族 | 右手 weld 腕转(烘 grip) |
| x | L 族 | 左手 weld 腕转(烘 grip) |
| x | M / M' | 左 / 右中指竖扫 |
| y | U | 右食指拨(勾弯横拉) |
| y | U' | 左食指拨;或 `U'p` 右食指推(§4.5) |
| y | D | 左无名指拨 |
| y | D' | 右无名指拨 |
| y | E 族 | 就近无名指 |
| z | F | 双中手:右食指越顶 topPush;左中右下:右食指下拨(§4.3);其它握:左拇指沿 F 面上扫 |
| z | F' | 双中手:左食指 topPush;其它握:右拇指上扫 |
| z | B / B' | 双中手 B':右食指(§4.4);其余:后侧中指竖扫 |
| z | S 族 | 后侧中指竖扫 |
| — | x / y / z 整体 | 双手 weld 刚体跟转 + 45° 踏移 |

180° 招式(U2 / F2 / D2' 等):命中 §4 连拨规格的走双指连拨;未命中的沿用单指 + softClamp(超 93° 部分压缩)。

## 4. 专项指法规格(用户口述规格)

### 4.0 接触强度分级

| 级别 | 含义 | 判据(oracle) |
|------|------|---------------|
| 贴块 pin | 指尖钉住**初始那一块**随层走到终点 | 计数回转后指尖钉锚点,肉垫距贴纸 ~1.2U |
| 贴面 contact | 指尖肉垫全程**接触魔方表面**(贴纸在指腹下滑动,块可以换) | 接触段 padGap ∈ [0.4, 2.5]U 带内 |
| 自由 free | 无接触要求(仅穿模禁令) | pen ≤ 0 |

### 4.1 双指连拨(通则,2026-07-08 用户规格)

一次 180° 转动由同手两根手指接力:

1. 第一指扫前 90°(层角 0 → 90°);
2. **第一指做完不恢复**——保持其 90° 末姿;
3. 第二指扫后 90°(90° → 180°);第二指的就位(预备)动作在前 90° 期间完成;
4. 第二指做完后,**两指一起恢复**回 home。

拖拽路径(不知道目标角度)第二指在层角越过 90° 时反应式接力;播放/键盘路径由 `prepareTwist(…, quarters)` 提示 180°,第二指提前预备。

| U' | 左食指拨(镜像贴面) | 贴面(推定,镜像) | 推定 |

| D2' | 连拨:**先右小指,后右无名指**(顺序按口述「小指和无名指」;若实际习惯反序,改本行 + classify 表一处) | 贴面 | 规格 2026-07-08 |

### 4.3 左中手 + 右下手(L home + R down)

| 招式 | 指法 | 接触 | 状态 |
|------|------|------|------|
| F | 右食指下拨(食指尖在 down 握天然位于 U 面 UFR 区,压着初始 UFR 往前下带) | 贴面 | 规格 2026-07-08 |
| F2 | 连拨:**先右食指做一个 F,再右中指做一个 F**(中指从 UR 棱滑前到 UF 缘接力) | 贴面 | 规格 2026-07-08 |
| 其它招式 | 沿用默认表 | — | — |

条件判定:严格要求 L grip = home 且 R grip = down;其它组合(双下手等)未定义,回落默认表。

### 4.4 双中手 B'(右食指)

- B' 时右食指从 BUR 位置拨 B 层(层动方向与 U 拨同族:勾弯横拉的 z 层版)。
- B(反向)推定左食指镜像。

### 4.5 U' 推法(记号 `U'p`)

1. 前置就位(转动开始**之前**,同 topPush 的 reach 节拍/闸步):右食指伸展,指尖放到 **RUB** 角块 U 面贴纸;
2. 推动 U 层逆时针 90°:指尖全程贴面,随初始 RUB 角块从 RUB 走到 **BLU**(=LUB);
3. 收指回 home。
- 实质是「贴块 pin」级(推着同一块走),推进方向 = 指尖沿 U 面从右后扫到左后。
- 仅播放/键盘路径可用(需要前置);拖拽不触发推法。
- **当前实现 v1 只达成前半程贴面**,后半段脱面滑行 —— 限制与升级路径见 §6。

## 5. 实现与验证索引

- 映射:`handsRig.ts` `classifyHandGesture(axis, cls, dir, grips, hint)`;测试锁定 `tests/hands_gestures.test.ts`。
- 连拨状态机:`HandState.flickFinger2` 通道(第一指保持 / 第二指接力 / 同步恢复)。
- 前置就位闸:`prepareTwist` / `isReachPreparing`;**收指闸** `isReachRetreating`(reach 式手势 decay+walk-back 未完不放行下一步,防 weld 把伸指烘着转)。两闸都在 `PlayerControls.gateHandsReach`。
- 跟随曲线:`HOOK_FOLLOW`(每手每键 cubic fit,θ 密采样 0.02~0.03 步长解 —— 方形层 45° 角部伸出 √2 半宽,粗采样会漏 ~0.1rad 的窄穿模窗口)。
- 避让通道:`THUMB_EVADE_D`(D 族静止拇指)、`BACK_EVADE`(B 族静止中/无名指)、legacy dodge(styled 手势的另一只手;upPush 也让)。
- 回撤:hook/backHook/downPush 的 decay 是「s 冻结 + fit 输出 ×decayK + RETREAT_LIFT 伸展凸包」——禁沿贴面曲线倒放(提交后层已复位,倒放中段穿层);upPush 沿 U 面弧退例外。U'p 就位/收指走 `UP_PUSH.transit` 转移凸包(直插会切上后棱)。
- 贴面/贴块判据与穿模 oracle、标定求解器:memory `project_sim_hands_rig`(两域 pen oracle + padGap 带)。
- 记号解析:`PlayerControls.tsx` `parseNxnItems`(`p` 后缀在 `GRIP_MARKS` 同层剥取)。

### 5.1 已知残差(2026-07-08 真实播放 oracle,全序列 worst −2.86U / 23 坏帧)

| 位置 | 深度 | 时长 | 性质 |
|------|------|------|------|
| U/U2/F/F2 起始 θ≈0.05 | −0.4~−0.6U | 1~4 帧 | 结构性:home 指腹本停在角部扫掠带内,cubic 在 s=0 无力;真实转动该角度一闪而过 |
| D2' 回撤 | −2.9U | ~3 帧 | 双指深绕缠的关节直线回撤弦切,RETREAT_LIFT 单通道极限;要清零需 per-style 回撤编排 |
| 各 hook 收尾/交接瞬态 | ≤0.5U | 1~6 帧 | 角部窄峰的动态余量,静态 0.03 步长已清 |

## 6. 未定规格(待用户补充,当前按「推定」列实现或回落默认)

- B(非 B')双中手指法:按 B' 镜像推定为左食指(已标定上线)。
- 双下手 / 双上手 / 左下右中 等其余握持组合的 F 族、连拨。
- **U2' / D2 镜像连拨**:L 侧双指 fit 未标定,当前回落单指 hook + softClamp;要启用需在 L 侧解 hook_middle2 / hook_pinky / hook_ring2 三组 fit 后放开 `classifyHandGesture` 的 `hand === "R"` 限制。
- `Up`(左食指推 U)编排。
- **U'p 全程贴块**:v1 只保证「就位压 RUB + 起推段跟随」,推行程后半段指尖脱面靠层滑行(RUB→BLU 弧长 ~190U 超出独臂可达域;裸腕偏航会把静止指排甩进魔方顶,需与四指避让联合编排)。
- 连拨记号是否需要显式标注(当前 180° 招式自动连拨,无需记号)。
