# /sim 手部指法规格(FINGERTRICKS)

**单一权威来源**:所有「哪只手、哪根手指、以什么方式做哪一步」的规格记在本文档。
改指法 → 先改这里 → 再改 `handsRig.ts` 的 `classifyHandGesture`(映射)与 `applyHand`(编排)→ 同步 `tests/hands_gestures.test.ts`。
穿模禁令 / oracle / 标定方法见 memory `project_sim_hands_rig`(不在本文档重复)。

坐标约定:标准配色正视,F=前(z+)、B=后(z−)、U=上(y+)、D=下(y−)、R=右(x+)、L=左(x−)。
角块记号如 UFR = U/F/R 三面交角。单位 U = 世界单位(SIZE=64 时棱长 192、半宽 96)。

## 0. 手指记号与总原则

### 0.1 手指记号(2026-07-09 约定)

前缀 L/R = 左/右手,编号 1~5 = 拇指/食指/中指/无名指/小指:

| 记号 | 手指 | 记号 | 手指 |
|------|------|------|------|
| L1 | 左手拇指 | R1 | 右手拇指 |
| L2 | 左手食指 | R2 | 右手食指 |
| L3 | 左手中指 | R3 | 右手中指 |
| L4 | 左手无名指 | R4 | 右手无名指 |
| L5 | 左手小指 | R5 | 右手小指 |

全文统一用此记号;`Lh` / `Rh` = 整手(仅 FTN §7 用)。

### 0.2 总原则(2026-07-09 用户规格)

**手作为一个整体,在任何时候不要远离魔方** —— 禁止整手平移 / 撤退式避让;单个手指可以短暂离面避让。

现状(r2,2026-07-09 六族迁移全部完成,`DODGE_MAG` 全表 0,整手平移全废):

| 族 | DODGE_MAG | 避让常量 | 避让指(先测后改,按实测受害标定) | 弹指腕借力 |
|----|-----------|----------|------------------------------------|------------|
| y.high(U) | 16→0(r1) | `HOOK_INDEX_EVADE` | 静止手食指沿 B 面外法线 −z 离面 | 0.1 保留 |
| y.low(D) | 44→0 | `LOW_EVADE` | 静止手无名指 −z 离面 + 拇指甩出 | 0.1 保留 |
| y.mid(E) | 62→0 | `MID_EVADE` | **双手**拇指甩出 + 中指 −z 离面;弹指手无名指扫掠加 −z 偏置 | 归零 |
| z.low(B) | 40→0 | `B_EVADE` | 静止手食指 + 中指 −z 离面 | 0.1 保留 |
| z.high(F) | 10→0 | `F_EVADE` | 按该手握姿选指:up=拇指甩出+无名指离 D 面;down=食指离 U 面;home 不动 | 归零 |
| z.mid(S) | 10→0 | `S_EVADE` | 按握姿选指:up/down=拇指甩出+中指离面;home 实测全程干净(旧 10 纯保险) | 归零 |

- 动作复用:拇指甩出 = `THUMB_EVADE_D` 同款(D/E/F/S 四族同一动作);食指离面 = `HOOK_INDEX_EVADE` 同款(U/B/F 统一);中指离面 = `MID_EVADE.middle` 同款(E/B/S 统一)。
- up/down 握 S 的旧 dodge=10 本就穿 6.7U,迁移顺带修复既有暴露。
- 未迁移仍整手 dodge 的族:无。

## 2. 记号汇总(解法框)

| 记号 | 含义 | 解析 |
|------|------|------|
| `↑` `↓` `·` | 换握(上手 / 下手 / 回中手),**手别由空白定**(见下) | 占一步,PlayerControls `GRIP_MARKS` |
| `p` 后缀 | **推法**(push):该步用「指尖推层」指法而非默认拨法 | 紧跟在招式 token 后,如 `U'p`;喂 cubing.js 前剥除,对魔方状态零作用 |
| `[...]` 注解块 | FTN 指法注解(手指 / 相序 / 路径 / 技法),紧贴招式 token | §7;喂 cubing.js 前整块剥除(§7.3),未实装 |

**换握手别规则(2026-07-10 用户规格)**:记号**紧贴**后续字符(无空格,如 `↑U`)= **右手**;记号后随**空白或串尾**(如 `↑ U`)= **左手**。
- `↓F` = 右手下手再 F(直达 §4.3 左中右下);`↓ F` = 左手下手再 F。
- 双手同换几乎不用;要写用紧贴链:`↑↑ U` = 第一个 `↑` 紧贴第二个(右手上),第二个后随空格(左手上)。
- 换握仍各占一步(`↑U` = 换握 + U 两步);紧贴只是词法上的手别标记,对招式解析零影响(剥除后 `↑U` → `U`)。
- **禁**在两招式之间无空格夹记号(`U·F`):剥除按「记号→空格」处理不会粘连,但可读性差,不要写。
- 空白是本记号系统唯一有语义的场合(§7.1 消歧规则的例外):**转写 / 格式化工具不得增删记号周边空格**。

**`p` 记号定义(2026-07-08 本文档发明)**:
- 语法:招式 token 尾接小写 `p`,与招式之间无空格(`U'p` ✓;`U' p` ✗)。助记:push / 推 / 平推。
- 当前仅定义 **`U'p`**:R2 推 U'(见 §4.5)。镜像扩展 `Up`(L2,LUB→RUB)保留但未编排。
- 其它招式带 `p` = 未定义:解析时剥掉后按默认指法播放(不报错),便于向后兼容将来扩展。
- 剥除函数与 `stripGripMarks` 同层处理:invert / mirror / 反推打乱 / recon 手递前都必须剥。
- `↑↓·` 与 `p` 是 FTN(§7)的糖形式,完整编码语法见 §7.1。

### 4.1 双指连拨(通则,2026-07-08 用户规格)

一次 180° 转动由同手两根手指接力:

1. 第一指扫前 90°(层角 0 → 90°);
2. **第一指做完不恢复**——保持其 90° 末姿;
3. 第二指扫后 90°(90° → 180°);第二指的就位(预备)动作在前 90° 期间完成;
4. 第二指做完后,**两指一起恢复**回 home。

拖拽路径(不知道目标角度)第二指在层角越过 90° 时反应式接力;播放/键盘路径由 `prepareTwist(…, quarters)` 提示 180°,第二指提前预备。

### 4.5 U' 推法(记号 `U'p`)

1. 前置就位(转动开始**之前**,同 topPush 的 reach 节拍/闸步):R2 伸展,指尖放到 **RUB** 角块 U 面贴纸;

### 4.6 weld 贴块钉指(FTN `@pin`,首例 `R[R2:@C>Q]`)

2. R weld 腕转:整手随 R 层刚体转,R2 钉的 UFR 就在转动层 —— 指块零相对运动,骑到 **Q**(= R2 的 home 指位 BUR,C 随 R 的落点被群论唯一决定,记谱终点可省:`R[R2:@C]` ≡ `R[R2:@C>Q]`);

## 7. 指法编码 FTN(2026-07-09 定稿)

招式层与编舞层物理分离:招式保持标准 WCA token 原样,指法全部关进紧跟招式的 `[...]` 注解块。
一切可缺省,缺省即回落 §3 默认表 —— 裸 `U` ≡ `U[R2:Q>M]`。`↑↓·` 与 `p` 后缀(§2)原样保留为糖:`U'p` ≡ `U'[R2:@B>A .p]`。

### 7.1 语法(BNF)

```
solution ::= step (SP step)*                            ; SP = 空格,顶层唯一分隔
step     ::= gripmark* (gripmark | move ["p"] ["[" clauses "]"])
                                                        ; "[" 紧贴招式,禁空格;gripmark 不带块
gripmark ::= "↑" | "↓" | "·"                            ; §1,占一步;手别由空白定(§2):
                                                        ; 紧贴后续字符=右手,后随 SP/串尾=左手
move     ::= 标准 WCA token
clauses  ::= clause (";" clause)*                       ; 括号内空白不敏感;块内禁 "]"
clause   ::= finger ":" [phase] body [tech]
body     ::= path                                       ; 扫掠
           | "~" [evade]                                ; 离场/避让(无 evade = 原地抬离)
finger   ::= ("L"|"R") ("1".."5" | "h")                 ; 1拇 2食 3中 4无名 5小 h整手
phase    ::= "(" num "-" num ")"                        ; 归一化转动进度 ⊆ [0,1],缺省 (0-1)
path     ::= [tier] waypoint (">" [tier] waypoint)*     ; tier 可在途中就地改档
tier     ::= "@" | "~"                                  ; 贴块 | 自由;缺省 = 贴面(§4.0)
waypoint ::= sticker [orbit [ring]]
sticker  ::= "A".."X"                                   ; Speffz,恒大写
orbit    ::= "c"|"e"|"w"|"x"|"t"|"m"|"o"                ; 角 棱 wing x心 +心 midge 斜;恒小写,缺省 c
ring     ::= digit                                      ; ≥5 阶同类轨道环号,必须跟在 orbit 后
tech     ::= ".f" | ".p" | ".t" | ".w"                  ; 拨 推 越顶 weld;缺省按 §3;只挂 clause 尾
evade    ::= dir [num]
dir      ::= ("+"|"-")("x"|"y"|"z") | face "^"          ; 轴向 | 面外法线
face     ::= "U"|"D"|"L"|"R"|"F"|"B"
num      ::= digit+ ["." digit+] | "." digit+           ; 距离单位 U;省略 = 取标定常量(如 HOOK_EXIT)
```

消歧三则:
1. `[` 与招式之间禁空格 —— 顶层只按空白分步,括号自定界,永不与招式序列打架。
2. 贴纸恒大写、轨道恒小写:`M`(角,c 缺省)/ `Me`(棱)/ `Mw`(wing),Speffz 跨轨重名免分隔符即消。
3. `~` 双位不双义:后随 `+`/`-` 或 `X^` 形或直接结束 = 离场 body;后随贴纸起始的 waypoint = 自由档 tier。
同一手指可出现多个 clause = 多段相序(接力 / 早退场);同一招多 clause 以 `;` 并列 = 并发。

### 7.3 剥离(喂 cubing.js 前)

```js
const CUBE_ONLY = s => s
  .replace(/\[[^\]]*\]/g, '')        // ① 剥注解块(先剥,U'p[…] 塌成 U'p)
  .replace(/[↑↓·]/g, ' ')            // ② 剥换握(换成空格:紧贴形 ↑U 塌成 " U",禁粘连;同 stripGripMarks)
  .replace(/(?<=\S)p(?=\s|$)/g, '')  // ③ 剥 push 糖(WCA token 无 p 结尾,安全)
  .replace(/\s+/g, ' ').trim();
```

与 §2 `stripGripMarks` / `p` 剥取同层;invert / mirror / 反推打乱 / recon 手递前统一走它。
验证:例 9 → `U2 U' D' F2 U'`;例 8 → `U F U' B'`。

### 7.4 为什么这样设计

招式层零污染是硬约束:注解块自定界,一条正则剥净,永不与招式空格混排。注解是纯叠加 —— 缺省全回落 §3/§4 既有表,写多少细多少,裸招式永远合法。大小写分工(贴纸大写/轨道小写)加缺省 c,根治 Speffz 跨轨重名且不付后缀噪声。相序 phase + 同指多段是唯一时序原语:接力、早退场、并发避让、pin→release 全由它派生,不另设同步/接力算子,符号表最小。`↑↓·` 与 `p` 是糖不是特例,旧记号零迁移成本;写错的注解剥离后招式照播,永不因编舞层损坏魔方层。换握手别的空白规则(§2)是全系统**唯一**空白敏感点(用户权衡:双手同换几乎不存在,单手常见形连后缀字母都不付)——代价是格式化工具不得动记号周边空格,且手别错写不报错只错编舞;剥离对两种形等价,魔方层始终免疫。
