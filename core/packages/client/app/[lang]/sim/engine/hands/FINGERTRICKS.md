# /sim 手部指法规格(FINGERTRICKS)

**单一权威来源**:所有「哪只手、哪根手指、以什么方式做哪一步」的规格记在本文档(2026-07-09 由旧文件名改为现名)。
改指法 → 先改这里 → 再改 `handsRig.ts` 的 `classifyHandGesture`(映射)与 `applyHand`(编排)→ 同步 `tests/hands_gestures.test.ts`。
穿模禁令 / oracle / 标定方法见 memory `project_sim_hands_rig`(不在本文档重复)。

坐标约定:标准配色正视,F=前(z+)、B=后(z−)、U=上(y+)、D=下(y−)、R=右(x+)、L=左(x−)。
角块记号如 UFR = U/F/R 三面交角。单位 U = 世界单位(SIZE=64 时棱长 192、半宽 96)。

## 0. 手指记号与总原则

### 0.1 手指记号(2026-07-09 约定)

前缀 L/R = 左/右手,编号 1~5 = 拇指/食指/中指/无名指/小指:

| 记号 | 手指 | 记号 | 手指 |
|------|------|------|------|
| L1 | 左手拇指 | R1 | 右手拇指 |
| L2 | 左手食指 | R2 | 右手食指 |
| L3 | 左手中指 | R3 | 右手中指 |
| L4 | 左手无名指 | R4 | 右手无名指 |
| L5 | 左手小指 | R5 | 右手小指 |

全文统一用此记号;`Lh` / `Rh` = 整手(仅 FTN §7 用)。

### 0.2 总原则(2026-07-09 用户规格)

**手作为一个整体,在任何时候不要远离魔方** —— 禁止整手平移 / 撤退式避让;单个手指可以短暂离面避让。

现状(r2,2026-07-09 六族迁移全部完成,`DODGE_MAG` 全表 0,整手平移全废):

| 族 | DODGE_MAG | 避让常量 | 避让指(先测后改,按实测受害标定) | 弹指腕借力 |
|----|-----------|----------|------------------------------------|------------|
| y.high(U) | 16→0(r1) | `HOOK_INDEX_EVADE` | 静止手食指沿 B 面外法线 −z 离面 | 0.1 保留 |
| y.low(D) | 44→0 | `LOW_EVADE` | 静止手无名指 −z 离面 + 拇指甩出 | 0.1 保留 |
| y.mid(E) | 62→0 | `MID_EVADE` | **双手**拇指甩出 + 中指 −z 离面;弹指手无名指扫掠加 −z 偏置 | 归零 |
| z.low(B) | 40→0 | `B_EVADE` | 静止手食指 + 中指 −z 离面 | 0.1 保留 |
| z.high(F) | 10→0 | `F_EVADE` | 按该手握姿选指:up=拇指甩出+无名指离 D 面;down=食指离 U 面;home 不动 | 归零 |
| z.mid(S) | 10→0 | `S_EVADE` | 按握姿选指:up/down=拇指甩出+中指离面;home 实测全程干净(旧 10 纯保险) | 归零 |

- up/down 握 S 的旧 dodge=10 本就穿 6.7U,迁移顺带修复既有暴露。
- 未迁移仍整手 dodge 的族:无。

## 1. 握持状态(grip)

| 名称 | GripName | 记号 | 定义 | 右手指位(左手 x 镜像) |
|------|----------|------|------|------------------------|
| 中手 | `home` | `·` | 基准握 | R1(右手拇指)压 FR 棱 F 面且**可见拇指肉整体落在 E 层带 y∈[−32,32]**(2026-07-09 用户规格,禁越入 U/D 带;解到 [−30.9,30.9],62U 肉径装 60U 窗的资产极限),R2(食指)BUR / R3(中指)BR / R4(无名指)BDR 贴 B 面,R5(小指)悬空收拢 |
| 上手 | `up` | `↑` | home 绕 x 轴 +90°(= R 提交后的腕姿) | R1 起手在 U 面,指列落 D 面 |
| 下手 | `down` | `↓` | home 绕 x 轴 −90° | R1 压 D 面(DR 缘),**R2 落 U 面 UFR 区 / R3 UR 棱 / R4 UBR 区**,指腹朝下压 U 面 |

- 记号占一步,只动手不动魔方(`regrip()` 动画 / `simulateGrips` 静态推演)。
- 组合名:**双中手** = 两手都 `home`;**左中右下** = 左 `home` + 右 `down`;其余类推。
- x 轴单层 weld(R/L 族)提交会把旋转烘进该手 grip(手停在转完的位置)——「上手」恰等于 home 被 R(或左手 L')提交后的姿态。

## 2. 记号汇总(解法框)

| 记号 | 含义 | 解析 |
|------|------|------|
| `↑` `↓` `·` | 换握(上手 / 下手 / 回中手) | 独立步,PlayerControls `GRIP_MARKS` |
| `p` 后缀 | **推法**(push):该步用「指尖推层」指法而非默认拨法 | 紧跟在招式 token 后,如 `U'p`;喂 cubing.js 前剥除,对魔方状态零作用 |
| `[...]` 注解块 | FTN 指法注解(手指 / 相序 / 路径 / 技法),紧贴招式 token | §7;喂 cubing.js 前整块剥除(§7.3),未实装 |

**`p` 记号定义(2026-07-08 本文档发明)**:
- 语法:招式 token 尾接小写 `p`,与招式之间无空格(`U'p` ✓;`U' p` ✗)。助记:push / 推 / 平推。
- 当前仅定义 **`U'p`**:R2 推 U'(见 §4.5)。镜像扩展 `Up`(L2,LUB→RUB)保留但未编排。
- 其它招式带 `p` = 未定义:解析时剥掉后按默认指法播放(不报错),便于向后兼容将来扩展。
- 剥除函数与 `stripGripMarks` 同层处理:invert / mirror / 反推打乱 / recon 手递前都必须剥。
- `↑↓·` 与 `p` 是 FTN(§7)的糖形式,完整编码语法见 §7.1。

## 4. 专项指法规格(用户口述规格)

### 4.0 接触强度分级

| 级别 | 含义 | 判据(oracle) |
|------|------|---------------|
| 贴块 pin | 指尖钉住**初始那一块**随层走到终点 | 计数回转后指尖钉锚点,肉垫距贴纸 ~1.2U |
| 贴面 contact | 指尖肉垫全程**接触魔方表面**(贴纸在指腹下滑动,块可以换) | 接触段 padGap ∈ [0.4, 2.5]U 带内 |
| 自由 free | 无接触要求(仅穿模禁令) | pen ≤ 0 |

### 4.1 双指连拨(通则,2026-07-08 用户规格)

一次 180° 转动由同手两根手指接力:

1. 第一指扫前 90°(层角 0 → 90°);
2. **第一指做完不恢复**——保持其 90° 末姿;
3. 第二指扫后 90°(90° → 180°);第二指的就位(预备)动作在前 90° 期间完成;
4. 第二指做完后,**两指一起恢复**回 home。

拖拽路径(不知道目标角度)第二指在层角越过 90° 时反应式接力;播放/键盘路径由 `prepareTwist(…, quarters)` 提示 180°,第二指提前预备。

### 4.4 双中手 B'(R2)

- B' 时 R2 从 BUR 位置拨 B 层(层动方向与 U 拨同族:勾弯横拉的 z 层版)。
- B(反向)推定 L2 镜像。

### 4.5 U' 推法(记号 `U'p`)

1. 前置就位(转动开始**之前**,同 topPush 的 reach 节拍/闸步):R2 伸展,指尖放到 **RUB** 角块 U 面贴纸;
2. 推动 U 层逆时针 90°:指尖全程贴面,随初始 RUB 角块从 RUB 走到 **BLU**(=LUB);
3. 收指回 home。
- 实质是「贴块 pin」级(推着同一块走),推进方向 = 指尖沿 U 面从右后扫到左后。
- 仅播放/键盘路径可用(需要前置);拖拽不触发推法。
- **当前实现 v1 只达成前半程贴面**,后半段脱面滑行 —— 限制与升级路径见 §6。

## 5. 实现与验证索引

- 映射:`handsRig.ts` `classifyHandGesture(axis, cls, dir, grips, hint)`;测试锁定 `tests/hands_gestures.test.ts`。
- 连拨状态机:`HandState.flickFinger2` 通道(第一指保持 / 第二指接力 / 同步恢复)。
- 前置就位闸:`prepareTwist` / `isReachPreparing`;**收指闸** `isReachRetreating`(reach 式手势 decay+walk-back 未完不放行下一步,防 weld 把伸指烘着转)。两闸都在 `PlayerControls.gateHandsReach`。
- 跟随曲线:`HOOK_FOLLOW`(每手每键 cubic fit,θ 密采样 0.02~0.03 步长解 —— 方形层 45° 角部伸出 √2 半宽,粗采样会漏 ~0.1rad 的窄穿模窗口)。2026-07-09 r3:`R.hook_index` / `R.hook_middle2` 全曲线 12 参数重解(节点参数化 v(1/3)/v(2/3)/v(1) 解耦中段与端点;**全顶点**穿透罚每 1.5° + 末端着陆窗 0.5° + 提交点本尊 —— 骨端扩壳近似与漏采样提交点两个坑都踩过;端点 = M-touch 接触目标,禁硬钉 3D 点,§4.2)。
- 单指避让:`HOOK_INDEX_EVADE`(R/L 同值 {c1:-0.07, c2:-0.04, c3:0, splay:0})—— U 族 y.high 静止手食指沿 B 面外法线(−z)微伸离面,2026-07-09 替代旧 `DODGE_MAG.y.high=16` 整手平移(§0.2 总原则;upPush 的静止手同走此通道)。通道 `HandState.indexEvade/indexEvadeTarget`:`tick` 每帧目标归零、即时跟升 / RECOVER_MS 300ms 回落(与 dodge 同时序);`driveGesture` 对 y.high flick(U/U'/U2/U'p)只写**另一只手**目标;`applyHand` 作用到该手食指关节(role===0 兜重叠帧);`rig.tuning.hookIndexEvade` 暴露标定。
- 单指避让通道族(r2,五个新常量,时序均同 `HOOK_INDEX_EVADE`:`min(1,|层角|/0.09)` 即时跟升 / RECOVER_MS 300ms 回落):`LOW_EVADE`(D 族,`HandState.lowEvade`,driveGesture 只写另一手)/ `MID_EVADE`(E 族,`midEvade`,写两手)/ `B_EVADE`(B 族,`bEvade`,写另一手;与 `BACK_EVADE` 分工 = 弹指手本侧中/无名指 vs 另一手食指/中指)/ `F_EVADE`(F 族,`fEvade`,写两手,applyHand 按该手握姿选指)/ `S_EVADE`(S 族,`sEvade`,写两手,握姿选指)。F/S 施加条件 `role===0 || flickDecay>0`(decay 段弹指也吃避让,防同类连发时上一步弹指回落被扫);标定经 `rig.tuning.{hookIndexEvade,lowEvade,midEvade,bEvade,fEvade,sEvade}`。
- 整手避让(legacy dodge):`DODGE_MAG` 全表 0(r2 六族迁移完成),表注释保留各族裸基线记录;`THUMB_EVADE_D`(D 族静止拇指)、`BACK_EVADE`(B 族弹指手本侧中/无名指)为单指通道,保留并被新常量复用数值。
- 弹指腕借力(0.1×层角):对 x / y.mid / z.high / z.mid 归零(贴面指半径 < 角柱半径时借力切向压入是实测穿模元凶);y.high / y.low / z.low 保留 0.1。
- 回撤:hook/backHook/downPush 的 decay 是「s 冻结 + fit 输出 ×decayK + RETREAT_LIFT 伸展凸包」——禁沿贴面曲线倒放(提交后层已复位,倒放中段穿层);upPush 沿 U 面弧退例外。U'p 就位/收指走 `UP_PUSH.transit` 转移凸包(直插会切上后棱)。**钉角 M 端姿(R 侧 hook_index / hook_middle2)走 per-style 覆盖 `RETREAT_LIFT_STYLED`**(§5.1 预留的 per-style 回撤编排,2026-07-09 r3 落地):含 splay 通道(+x 甩离 R 面是钉角端姿的天然逃逸向)+ **攻窗保持**(decayK ∈ [0.88,1] 段 fit 输出钳提交姿 kFit=min(1,decayK/0.88),凸包升满再衰减 —— 贴块姿 pad 离贴纸只 ~1.4U,×0.99 第一帧就拉进贴纸);其余样式回落默认 RETREAT_LIFT,路径逐位不变。
- U2 首指退场:`HOOK_EXIT`(role1+isDouble+hook+index 专属通道,{c1:0.139, c2:0.225, c3:-0.016, splay:-0.1})× `sm(min(1,sRaw2/HOOK_EXIT_IN))*kFit`,`HOOK_EXIT_IN=0.2`(Q2 前 20% 内到位);幅度由「Q2 全程全顶点 pen ≤0 + 最小幅度正则」解出(甩出扫掠环带,§4.2)。

## 7. 指法编码 FTN(2026-07-09 定稿)

招式层与编舞层物理分离:招式保持标准 WCA token 原样,指法全部关进紧跟招式的 `[...]` 注解块。
一切可缺省,缺省即回落 §3 默认表 —— 裸 `U` ≡ `U[R2:Q>M]`。`↑↓·` 与 `p` 后缀(§2)原样保留为糖:`U'p` ≡ `U'[R2:@B>A .p]`。

### 7.1 语法(BNF)

```
solution ::= step (SP step)*                            ; SP = 空格,顶层唯一分隔
step     ::= gripmark | move ["p"] ["[" clauses "]"]    ; "[" 紧贴招式,禁空格;gripmark 不带块
gripmark ::= "↑" | "↓" | "·"                            ; §1,独立步
move     ::= 标准 WCA token
clauses  ::= clause (";" clause)*                       ; 括号内空白不敏感;块内禁 "]"
clause   ::= finger ":" [phase] body [tech]
body     ::= path                                       ; 扫掠
           | "~" [evade]                                ; 离场/避让(无 evade = 原地抬离)
finger   ::= ("L"|"R") ("1".."5" | "h")                 ; 1拇 2食 3中 4无名 5小 h整手
phase    ::= "(" num "-" num ")"                        ; 归一化转动进度 ⊆ [0,1],缺省 (0-1)
path     ::= [tier] waypoint (">" [tier] waypoint)*     ; tier 可在途中就地改档
tier     ::= "@" | "~"                                  ; 贴块 | 自由;缺省 = 贴面(§4.0)
waypoint ::= sticker [orbit [ring]]
sticker  ::= "A".."X"                                   ; Speffz,恒大写
orbit    ::= "c"|"e"|"w"|"x"|"t"|"m"|"o"                ; 角 棱 wing x心 +心 midge 斜;恒小写,缺省 c
ring     ::= digit                                      ; ≥5 阶同类轨道环号,必须跟在 orbit 后
tech     ::= ".f" | ".p" | ".t" | ".w"                  ; 拨 推 越顶 weld;缺省按 §3;只挂 clause 尾
evade    ::= dir [num]
dir      ::= ("+"|"-")("x"|"y"|"z") | face "^"          ; 轴向 | 面外法线
face     ::= "U"|"D"|"L"|"R"|"F"|"B"
num      ::= digit+ ["." digit+] | "." digit+           ; 距离单位 U;省略 = 取标定常量(如 HOOK_EXIT)
```

消歧三则:
1. `[` 与招式之间禁空格 —— 顶层只按空白分步,括号自定界,永不与招式序列打架。
2. 贴纸恒大写、轨道恒小写:`M`(角,c 缺省)/ `Me`(棱)/ `Mw`(wing),Speffz 跨轨重名免分隔符即消。
3. `~` 双位不双义:后随 `+`/`-` 或 `X^` 形或直接结束 = 离场 body;后随贴纸起始的 waypoint = 自由档 tier。
同一手指可出现多个 clause = 多段相序(接力 / 早退场);同一招多 clause 以 `;` 并列 = 并发。

### 7.3 剥离(喂 cubing.js 前)

```js
const CUBE_ONLY = s => s
  .replace(/\[[^\]]*\]/g, '')        // ① 剥注解块(先剥,U'p[…] 塌成 U'p)
  .replace(/[↑↓·]/g, '')             // ② 剥换握
  .replace(/(?<=\S)p(?=\s|$)/g, '')  // ③ 剥 push 糖(WCA token 无 p 结尾,安全)
  .replace(/\s+/g, ' ').trim();
```

与 §2 `stripGripMarks` / `p` 剥取同层;invert / mirror / 反推打乱 / recon 手递前统一走它。
验证:例 9 → `U2 U' D' F2 U'`;例 8 → `U F U' B'`。

### 7.4 为什么这样设计

招式层零污染是硬约束:注解块自定界,一条正则剥净,永不与招式空格混排。注解是纯叠加 —— 缺省全回落 §3/§4 既有表,写多少细多少,裸招式永远合法。大小写分工(贴纸大写/轨道小写)加缺省 c,根治 Speffz 跨轨重名且不付后缀噪声。相序 phase + 同指多段是唯一时序原语:接力、早退场、并发避让、pin→release 全由它派生,不另设同步/接力算子,符号表最小。`↑↓·` 与 `p` 是糖不是特例,旧记号零迁移成本;写错的注解剥离后招式照播,永不因编舞层损坏魔方层。
